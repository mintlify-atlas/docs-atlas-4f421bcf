## Prerequisites

Before you start, make sure you have:

- **Node.js >=22.12.0** — check with `node -v`
- **npm** — bundled with Node.js
- A running **backend API** with a base URL (e.g. `https://api.yourdomain.com`)
- A **PostgreSQL** database seeded with the Academia Gema schema

<Info>
  The frontend is a pure client-side SPA. It does not include a backend or database — you must supply those separately.
</Info>

## Steps

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/LincolmVF/AcademiaGema.git
    cd AcademiaGema/Front
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```

    This installs all production and development dependencies declared in `package.json`, including React 19, React Router v7, Tailwind CSS v3, Recharts, and Framer Motion.
  </Step>

  <Step title="Configure the environment variable">
    Create a `.env` file in the `Front/` directory:

    ```bash
    # Front/.env
    VITE_API_URL=https://your-backend-api-url.com
    ```

    Replace the value with the base URL of your deployed REST API. All API calls are prefixed with this value at runtime.

    <Warning>
      Do not include a trailing slash in `VITE_API_URL`. For example, use `https://api.example.com`, not `https://api.example.com/`.
    </Warning>
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run dev
    ```

    Vite starts a local server with Hot Module Replacement. Open the URL printed in your terminal (typically `http://localhost:5173`).
  </Step>

  <Step title="Log in">
    Navigate to `http://localhost:5173/login`. Enter the email and password for the account you want to test.

    After a successful login, the server sets cookies (`user_role`, `user_name`, `user_lastname`, `user_id`) and redirects you to the dashboard that matches your role.
  </Step>
</Steps>

## First actions per role

<Tabs>
  <Tab title="Administrador">
    After logging in you land on `/dashboard/admin` — the overview dashboard with platform-wide statistics.

    **Recommended first steps:**

    1. Go to **Sedes** (`/dashboard/admin/locations`) and create at least one location and court.
    2. Go to **Niveles** (`/dashboard/admin/levels`) and add your training levels with monthly prices.
    3. Go to **Horarios** (`/dashboard/admin/schedule`) and build your weekly class schedule.
    4. Go to **Alumnos** (`/dashboard/admin/students`) to register your first students.
    5. Go to **Validación de pagos** (`/dashboard/admin/payment-validation`) to review any pending payment reports.
  </Tab>

  <Tab title="Alumno">
    After logging in you land on `/dashboard/student` — your personal training hub.

    **Recommended first steps:**

    1. Go to **Inscripción** (`/dashboard/student/enrollment`) to browse available class slots and enroll in one.
    2. Go to **Mis inscripciones** (`/dashboard/student/myRegistrations`) to confirm your enrollment is active.
    3. Go to **Pagos** (`/dashboard/student/payments`) to view your pending accounts and report a payment once you have made one.
  </Tab>

  <Tab title="Coordinador">
    After logging in you land on `/dashboard/teacher` — the attendance management view.

    **Recommended first steps:**

    1. Select the class session for today from the schedule list and mark attendance for each enrolled student.
    2. Go to **Perfil** (`/dashboard/teacher/profile`) to review and update your contact information.
    3. Go to **Día de corte** (`/dashboard/teacher/DiaCorte`) to review the billing-cycle summary for your classes.
  </Tab>
</Tabs>
