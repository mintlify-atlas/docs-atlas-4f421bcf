---
title: Payments API
description: Endpoints for reporting and validating payments, managing accounts receivable, and accessing the pricing catalog.
---

Payment endpoints cover the full financial cycle: students report payments, admins validate them, and accounts receivable tracks what is owed.

---

## GET /pagos/gestion-admin

Returns the full list of payments for admin review, including pending validations.

**Authentication:** Required (admin).

### Response

<ResponseField name="success" type="boolean" required />
<ResponseField name="data" type="object[]">
  <Expandable title="payment object">
    <ResponseField name="id" type="number">Payment ID.</ResponseField>
    <ResponseField name="cuenta_id" type="number">Account receivable ID this payment applies to.</ResponseField>
    <ResponseField name="monto_pagado" type="number">Amount paid.</ResponseField>
    <ResponseField name="fecha_pago" type="string">ISO 8601 payment timestamp.</ResponseField>
    <ResponseField name="metodo_pago" type="string">Payment method string.</ResponseField>
    <ResponseField name="referencia_transaccion" type="string">Transaction reference or voucher number.</ResponseField>
  </Expandable>
</ResponseField>

## POST /pagos/reportar

Allows a student to report a payment they have made. The payment is created in a pending state until an admin validates it.

**Authentication:** Required (student).

### Request body

<ParamField body="cuenta_id" type="number" required>
  The accounts receivable ID (`cuentas_por_cobrar.id`) this payment covers.
</ParamField>

<ParamField body="monto_pagado" type="number" required>
  The amount paid, as a decimal (e.g. `150.00`).
</ParamField>

<ParamField body="metodo_pago" type="string" required>
  Payment method used (e.g. `Transferencia`, `Efectivo`, `Yape`).
</ParamField>

<ParamField body="referencia_transaccion" type="string">
  Transaction reference number or voucher code.
</ParamField>

### Response

<ResponseField name="success" type="boolean" required />
<ResponseField name="data" type="object">
  The reported payment record in pending state.
</ResponseField>

---

## POST /pagos/validar

Allows an admin to approve or reject a reported payment.

**Authentication:** Required (admin).

### Request body

<ParamField body="pago_id" type="number" required>
  The payment ID to validate.
</ParamField>

<ParamField body="action" type="string" required>
  The validation decision. Must be `approve` or `reject`.
</ParamField>

### Response

<ResponseField name="success" type="boolean" required />
<ResponseField name="data" type="object">
  The updated payment record.
</ResponseField>

---

## GET /cuentas-por-cobrar

Returns the full accounts receivable list. Used by admins to see outstanding balances.

**Authentication:** Required (admin).

### Response

<ResponseField name="success" type="boolean" required />
<ResponseField name="data" type="object[]">
  <Expandable title="account object">
    <ResponseField name="id" type="number">Account ID.</ResponseField>
    <ResponseField name="alumno_id" type="number">Student user ID this account belongs to.</ResponseField>
    <ResponseField name="concepto" type="string">Description of the charge (e.g. `Mensualidad Julio 2025`).</ResponseField>
    <ResponseField name="monto" type="number">Total amount charged.</ResponseField>
    <ResponseField name="fecha_vencimiento" type="string">Due date in `YYYY-MM-DD` format.</ResponseField>
    <ResponseField name="estado" type="string">`PENDIENTE`, `PARCIAL`, `PAGADO`, `VENCIDO`, or `CANCELADO`.</ResponseField>
    <ResponseField name="creado_en" type="string">ISO 8601 creation timestamp.</ResponseField>
    <ResponseField name="actualizado_en" type="string">ISO 8601 last-update timestamp.</ResponseField>
  </Expandable>
</ResponseField>

## POST /cuentas-por-cobrar

Creates a new account receivable entry for a student. Used by admins to generate monthly charges.

**Authentication:** Required (admin).

### Request body

<ParamField body="alumno_id" type="number" required>
  The student's user ID.
</ParamField>

<ParamField body="concepto" type="string" required>
  Description of the charge (max 150 characters).
</ParamField>

<ParamField body="monto" type="number" required>
  Amount to charge as a decimal.
</ParamField>

<ParamField body="fecha_vencimiento" type="string" required>
  Due date in `YYYY-MM-DD` format.
</ParamField>

### Response

<ResponseField name="success" type="boolean" required />
<ResponseField name="data" type="object">
  The newly created account receivable.
</ResponseField>
