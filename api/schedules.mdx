---
title: Schedules API
description: Endpoints for managing class schedules, training levels, mass rescheduling operations, and holidays.
---

Schedule endpoints manage `horarios_clases`, `niveles_entrenamiento`, class session operations, and the `feriados` (holiday) calendar. Most write operations require an admin or coordinator role.

---

## GET /horarios

Returns all class schedules. This is a public endpoint — no authentication is required.

### Response

<ResponseField name="success" type="boolean" required />

<ResponseField name="data" type="object[]">
  <Expandable title="schedule object">
    <ResponseField name="id" type="number">Schedule ID.</ResponseField>
    <ResponseField name="sede_id" type="number">Venue ID.</ResponseField>
    <ResponseField name="cancha_id" type="number">Court ID.</ResponseField>
    <ResponseField name="profesor_id" type="number">Professor user ID.</ResponseField>
    <ResponseField name="nivel_id" type="number">Training level ID.</ResponseField>
    <ResponseField name="dia_semana" type="number">Day of week: `1` = Monday … `7` = Sunday.</ResponseField>
    <ResponseField name="hora_inicio" type="string">Start time in `HH:MM:SS` format.</ResponseField>
    <ResponseField name="hora_fin" type="string">End time in `HH:MM:SS` format.</ResponseField>
    <ResponseField name="capacidad_max" type="number">Maximum enrolled students. Default is `20`.</ResponseField>
    <ResponseField name="activo" type="boolean">Whether the schedule is currently active.</ResponseField>
  </Expandable>
</ResponseField>

## POST /horarios

Creates a new class schedule. Admin only.

**Authentication:** Required (admin).

### Request body

<ParamField body="sede_id" type="number" required>
  The venue (`sedes`) where the class takes place.
</ParamField>

<ParamField body="cancha_id" type="number" required>
  The court (`canchas`) assigned to this schedule.
</ParamField>

<ParamField body="profesor_id" type="number" required>
  The professor's user ID (`profesores.usuario_id`).
</ParamField>

<ParamField body="nivel_id" type="number" required>
  The training level (`niveles_entrenamiento`) for this schedule.
</ParamField>

<ParamField body="dia_semana" type="number" required>
  Day of the week. `1` = Monday, `7` = Sunday.
</ParamField>

<ParamField body="hora_inicio" type="string" required>
  Start time in `HH:MM` format.
</ParamField>

<ParamField body="hora_fin" type="string" required>
  End time in `HH:MM` format.
</ParamField>

<ParamField body="capacidad_max" type="number" default="20">
  Maximum number of students. Defaults to `20`.
</ParamField>

<ParamField body="activo" type="boolean" default="true">
  Whether the schedule is active immediately.
</ParamField>

<Warning>
  The `(cancha_id, dia_semana, hora_inicio)` combination must be unique. The database enforces this with the `unica_cancha_hora` constraint.
</Warning>

### Response

<ResponseField name="success" type="boolean" required />
<ResponseField name="data" type="object">
  The newly created schedule object.
</ResponseField>

## POST /clases/reprogramar-masivo

Mass-reschedules all classes from one date to another. Used when a date needs to be moved globally (e.g. due to an event).

**Authentication:** Required (admin or coordinator).

### Request body

<ParamField body="fecha_original" type="string" required>
  The original date of the classes to move, in `YYYY-MM-DD` format.
</ParamField>

<ParamField body="fecha_nueva" type="string" required>
  The new date to reschedule the classes to, in `YYYY-MM-DD` format.
</ParamField>

### Response

<ResponseField name="success" type="boolean" required />
<ResponseField name="data" type="object">
  Summary of the reschedule operation.
</ResponseField>

---

## POST /clases/revertir-masivo

Reverts a previously applied mass reschedule, restoring classes to their original date.

**Authentication:** Required (admin or coordinator).

### Request body

<ParamField body="fecha_original" type="string" required>
  The original date of the classes before they were moved.
</ParamField>

<ParamField body="fecha_nueva" type="string" required>
  The date the classes were moved to (used to identify the reschedule to revert).
</ParamField>

### Response

<ResponseField name="success" type="boolean" required />
<ResponseField name="message" type="string">
  Confirmation message.
</ResponseField>
